SAL, the Semantic Accessibility Layer CLI, turns RDF source files into versioned data products.
It is designed for projects where the source data lives in Git, the built output is an Apache Iceberg table, and published artifacts need to preserve the link back to the exact source commit.

## Core workflow

1. Run [`sal init`](/reference/commands/#sal-init) in a Git repository with a configured remote.
2. Add RDF source files to the project.
3. Run [`sal validate`](/reference/commands/#sal-validate) while editing to catch invalid or undefined terms.
4. Commit source changes, then run [`sal build`](/reference/commands/#sal-build) to create or update the local data product.
5. Use [`sal query`](/reference/commands/#sal-query) to inspect the built table.
6. Publish the result with [`sal push`](/reference/commands/#sal-push) or [`sal upload`](/reference/commands/#sal-upload).

For command flags and examples, see the [command reference](/reference/commands/).
